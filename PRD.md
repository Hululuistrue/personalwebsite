# 个人网站 PRD（v1）

## 1. 文档信息
- 产品名称：个人品牌网站（Personal Website）
- 版本：v1.0
- 日期：2026-03-01
- 负责人：George Wang

## 2. 背景与目标
- 背景：需要一个“超级炫酷”的个人网站，用于展示个人能力、项目作品和合作入口。
- 核心目标：
  - 在 5 秒内建立强烈视觉记忆点。
  - 清晰传达“我是谁、我做过什么、如何联系我”。
  - 支持后续快速更新项目内容。

## 3. 成功指标（MVP）
- 首屏加载时间（LCP）目标：<= 2.5s（常见宽带环境）。
- 访问者到“联系动作”（点击邮箱/社媒）转化率：>= 5%。
- 移动端可用性：主要页面元素无重叠、无横向滚动。

## 4. 用户画像
- 潜在客户：看重案例质量、交付能力、沟通效率。
- 招聘方/合作方：看重技术栈、项目结果、成长潜力。
- 同行开发者：看重实现细节、交互和代码品质。

## 5. 范围定义
- In Scope（本期做）：
  - 单页官网：Hero / About / Projects / Contact。
  - 高识别视觉风格与轻量动画。
  - 响应式布局（桌面 + 手机）。
  - 基础 SEO（title、description、语义结构）。
- Out of Scope（本期不做）：
  - CMS 后台管理。
  - 多语言切换系统。
  - 登录、评论、复杂数据后台。

## 6. 信息架构
- Header：Logo、锚点导航。
- Hero：身份定位、核心价值、CTA。
- About：能力摘要、工作方式、关键数据。
- Projects：3-6 个项目卡片（标题、简介、链接）。
- Contact：邮箱/社媒/合作引导。
- Footer：版权信息。

## 7. 功能需求
## 7.1 Must Have
- F-01：导航锚点平滑滚动。
- F-02：首屏高冲击视觉（大标题、背景氛围、主按钮）。
- F-03：区块滚动显现动画（reveal）。
- F-04：项目卡片悬停反馈。
- F-05：移动端自适应。
- F-06：联系入口可直接点击（mailto / 社媒）。

## 7.2 Should Have
- F-07：背景交互效果（鼠标轻微联动）。
- F-08：连续视觉元素（如 ticker 跑马灯）增强节奏感。
- F-09：尊重 `prefers-reduced-motion` 的降级方案。

## 7.3 Could Have（后续）
- F-10：项目筛选（按类型/技术）。
- F-11：深色/浅色主题切换。
- F-12：博客或长文系统。

## 8. 视觉与交互规范
- 风格关键词：未来感、速度感、实验性、但保持可读性。
- 字体策略：展示字体 + 正文字体分离，避免默认系统感。
- 配色策略：高对比主色 + 辅色，背景使用渐变/纹理层。
- 动效策略：
  - 页面初载入：分层淡入；
  - 滚动：区块 reveal；
  - 悬停：卡片轻抬升+光感反馈；
  - 控制动效强度，避免晕动。

## 9. 内容策略
- 文案语气：简洁、自信、结果导向。
- 项目描述模板：问题 -> 方案 -> 结果（尽量量化）。
- 联系区文案：明确“可合作事项”和“响应速度”。

## 10. 技术方案
- 技术栈：`HTML + CSS + JavaScript`（静态站点，便于快速发布）。
- 性能：
  - 图片尽量压缩为 `webp/avif`。
  - 动效优先使用 `transform/opacity`。
- 可维护性：
  - 样式变量集中在 `:root`。
  - 区块按模块命名，避免样式冲突。

## 11. 验收标准（DoD）
- 在桌面和手机端，页面结构完整、可读、可点击。
- 所有导航锚点可达且定位准确。
- 动效在主流浏览器可运行；`reduced-motion` 下无关键体验损失。
- 联系方式可触发邮件客户端或跳转对应链接。
- Lighthouse（Performance/Best Practices/SEO）建议均达到可接受水平（非强制满分）。

## 12. 里程碑
- M1（已完成）：MVP 视觉版落地。
- M2：替换为真实个人资料和项目内容。
- M3：性能优化 + SEO 细化 + 部署上线（Vercel/Netlify/GitHub Pages）。
